## Cómo mantener esta documentación

1. **Todo cambio de rumbo se refleja aquí en el mismo commit que lo implementa.**
   Un documento desactualizado es peor que no tenerlo.
2. **`DECISIONES.md` es append-only**: no se reescribe el pasado; se añade.
3. **`ARQUITECTURA.md` es vivo**: se edita para reflejar el estado real del diseño.
4. Al añadir un documento nuevo, **enlázalo en esta tabla** y en `CLAUDE.md`.
